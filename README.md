# ArchiveSys (MVP+)

مشروع فعلي لتطبيق سطح مكتب محلي للأرشفة والفهرسة والبحث الشامل داخل المجلدات، مع واجهات عربية.

## ما تم بناؤه الآن

- واجهة PySide6 عربية (RTL) بأربع تبويبات:
  - **البحث**: استعلام + فلترة حسب الامتداد والمصدر + نتائج + معاينة.
  - **المصادر**: اختيار مجلد، تحديد recursive، تشغيل الفهرسة، عرض المصادر المسجلة.
  - **الوظائف والسجل**: عرض آخر مهام الفهرسة وآخر الأخطاء.
  - **لوحة التحكم**: KPIs تشغيلية مباشرة (مصادر/ملفات/وظائف/أخطاء).
- قاعدة SQLite محلية مع FTS5 للبحث النصي.
- فهرسة أولية للمجلدات:
  - حفظ metadata (المسار، الاسم، الامتداد، النوع، الحجم، التواريخ، SHA-256).
  - استخراج نص للأنواع النصية الشائعة.
  - إدراج الملفات غير المدعومة كعناصر قابلة للبحث بالاسم/المسار.
- تتبع Job لكل عملية فهرسة وتسجيل أخطاء القراءة.
- فهرسة غير متزامنة عبر QThreadPool مع شريط تقدم حي وزر إيقاف للمهمة الجارية.
- نواة Plugins/Extractors (Registry + PlainText extractor + fallback metadata-only).

## التشغيل

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m app.main
```

## الحالة الحالية

- النسخة الحالية **احترافية كأساس معماري MVP+** لكنها ليست النسخة النهائية المتقدمة جدًا بعد.
- ما زال مطلوبًا للإصدار الكامل:
  - OCR (Tesseract)
  - Extractors متخصصة لـ PDF/Office
  - Connectors لقواعد بيانات إضافية
  - تحسين إدارة الطوابير (pause/resume متعدد المهام) بدل إيقاف مهمة واحدة


## المنجز والمتبقي (مختصر تنفيذي)

- تم إنجاز نواة تطبيق عربية RTL مع فهرسة وبحث FTS5 وإدارة مصادر وسجل وظائف/أخطاء وDashboard مؤشرات.
- المتبقي للوصول لنسخة enterprise أكثر: OCR، Extractors متقدمة (PDF/Office/Tika)، Workers غير متزامنة بالكامل، وطبقة Plugins/Connectors.
